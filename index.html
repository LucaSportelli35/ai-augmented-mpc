Project Overview

Autonomous rendezvous and proximity operations in adversarial orbital environments require a controller that can pursue a target, avoid unsafe proximity with an adversary, and adapt online without sacrificing interpretability. This work proposes an AI-augmented Model Predictive Control (MPC) framework for a three-spacecraft interaction: the pursuing spacecraft Bandit, the target Lady, and the adversarial spacecraft Guard.

The core principle is separation of roles: the adaptive layer does not directly command thrust. Instead, it modifies interpretable optimization-level quantities such as tracking weights, safety penalties, prediction horizons, terminal weights, and keep-out-zone objectives. The thrust command is still produced by a constrained MPC problem with orbital dynamics, actuator limits, predictive safety constraints, and slack-based feasibility handling.

Method

Learning supervises the controller. MPC still commands the spacecraft.

1. Relative orbital dynamics

The prediction model uses Clohessy-Wiltshire linearized relative dynamics in the LVLH frame, enabling efficient real-time finite-horizon optimization.

$\dot{x}(t)=A_cx(t)+B_cu(t)$

2. Constrained MPC inner loop

At each control update, a parametric quadratic program is solved and only the first optimized control action is applied in receding-horizon fashion.

$u_k=z^{\star}_{1:3}(x_k,P_k)$

3. Adaptive supervisory layer

Offline derivative-free search and online tabular reinforcement learning select bounded MPC parameter updates based on the evolving orbital geometry.

$Q[s_k,a_k]\leftarrow Q[s_k,a_k]+\alpha(G_k-Q[s_k,a_k])$

Key design choice

The AI layer changes the optimization problem, not the thrust command.

This preserves a safety-aware, constraint-aware, and interpretable MPC structure while still allowing the controller to adapt its behavior online during dynamically evolving adversarial encounters.

Results

Monte Carlo robustness in adversarial RPO

The adaptive MPC controller was evaluated in the Kerbal Space Program Differential Game Capture-the-Satellite environment across a 1000-run Monte Carlo campaign.

Controller Comparison

Naive MPC vs fixed-parameter MPC vs adaptive MPC

Representative 25-run comparisons show that online supervisory adaptation produces more flexible behavior under aggressive Guard-interception geometries than fixed-parameter configurations.

Interpretability

Adaptive behavior can be traced through MPC parameters.

Since the supervisory layer acts on physically meaningful optimization parameters, its behavior can be connected to orbital geometry, keep-out-zone activity, and safety-performance tradeoffs.

Main Contributions

  • AI-augmented constrained MPC architecture for adversarial spacecraft rendezvous and proximity operations.
  • Learning-based parameter adaptation operating on optimization-level MPC quantities instead of direct thrust commands.
  • Integration of CW orbital dynamics, actuator limits, predictive keep-out-zone constraints, slack-based feasibility handling, and optional CBF safety filtering.
  • Monte Carlo validation showing improved adaptive behavior and robustness relative to representative fixed-parameter MPC baselines.
  • Interpretable autonomy framework where adaptive decisions can be related to safety metrics, MPC parameters, and orbital geometry.
